# HOWTO

## Предпосылки
### Проблема
#### Корпораций
- Нет единого представления происходящих процессов в компании (Уход ключевых сотрудников провоцирует вынужденный пересмотр процесса);
- Нет понимания истории принятия решений;
- Из-за распределённого найма и непрозрачных процессов копится большой процент неэффективных сотрудников;
- Этап онбординга палит несколько месячных бюджета;
- Нет возможности привлечь лояльных пользователей к учлучшению продукта;

- Невозможно бесшовно и органично интегрировать внешние ресурсы в продуктовый процесс;(?)

#### Спецов
- Большой процент синтетических задач в холостую (учебные, тестовые, ...);
- Решение похожих задач в разных компаниях (киты, библиотеки, ...);
- Отстутствие опенсорса по многим направления (редакторы и дизайнеры);

- Несколько паралелых комьюнити не имеющих точек соприкосновения (дизайнерское, редаткорское);(?)


## Решение
- Отображаем все процессы происходящии в компании простым способом;
- Возможность вовлекать лояльных ползователей в развитеие продукта;
- Возможность поучавствовать в развитии продукта или проспонсировать её выполнение;
- Возможность выполнить боевую задачу находясь вне компани и получить ганорар за выполнение;
- Поиск подходящей задачи каждому сотруднику (на которой он будет максимально эффективен);

- Единая и доступная точка входя для выполнения задач;

- Мы помогаем покрыть критический дефицит замкнув друг-на-друга остаточные ресурсы спецов на дефицит компаний [?]

## Спецификация
### Методология
**Принципы**
- Распределенность (принимается вклад любого масштаба);
- Иттеративность (у любого проекта всегда есть зазор для улучшения).

**Точки приложения силы**
- Заказчик (первичный набросок);
- Архитектор (выстраивание узлов);
- Исполнитель (рализация);

**Основные составлющие**
- Человек (создатель);
- Алгоритм (исполнитель);
- Устройство (инструмент).

**Процесс**
Результирующий узел формируется путём преобразования в рецепте входящих узлов

**Терминология**
- Узел (в разном контексте бывает входящим и результирующим);
- Рецепт (инсрукция преобразования входящих узлов);
- Шнур (содержит информацию о связе между узлами);
- Скоуп (смысловая область от входящих до результирующего узла);
- Кофициент[?] (количество затронутых Скоупов которые нужно учитывать для формирования результирующего узла);
- Генерация (выполнение рецепта);
- Регенерация (генерация с новыми входящими узлами);
- Цикл (время необходимое для выполнения одной генерации)
- Чейнинг (комбинация узлов);


**Настройка рецепта**
- Дифецит (отрицательная разница между потребляемыми и создаваемыми узлами);
- Профицит (положительная разница между потребляемыми и создаваемыми узлами).



### Этапы реализации
- Преза и макеты всего продукта;
- Продакшн: доски (создать карточки вручную или скормить ссылку);
- Прототип: нодовый редактор с простраиванием свзяи между узлами.
- Координата задачи в рецепте (штора с нодой);


### Социальные платформы
- Twitter как регулярный контакт с целевой аудиторией;
- Medium как профильный паблик с инструкциями.
